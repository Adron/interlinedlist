# Settings

Access your settings from the user dropdown (top right) → **Settings**.

## Profile Settings

- **Display Name** — How you appear to others (defaults to your username if not set)
- **Bio** — Short description shown on your public profile
- **Avatar** — Profile picture; upload a file or set from a URL
- **Theme** — Light, dark, or system (follows your device preference)
- **Max message length** — Character limit per message (default 666)

## View Preferences

- **Messages per page** — How many messages to load at once (10–30)
- **Viewing preference** — Show My Messages, All Messages, Followers Only, or Following Only in your Home feed
- **Show link previews** — Toggle link preview cards on or off

## Message Settings

- **Default message visibility** — Whether new messages default to public or private
- **Advanced post settings** — Show the gear icon next to the message input so you can attach images, video, and cross-post when composing

## Profile Location

- **Latitude / Longitude** — Optional location for your profile, used by the Weather widget and similar location-aware features

## Permissions

- **Email visibility** — Control who can see your email address
- **Account visibility** — Make your account private; new followers will need to request to follow you and you must approve them

## Security

### Password

- **Change password** — Enter your current password and a new one to update your credentials

### Connected Accounts

Link social and developer accounts for sign-in and cross-posting. Go to **Settings → Security** (or the Connected Accounts section) to link or unlink any of the following:

- **GitHub** — Sign in with your GitHub account. If you want to use GitHub-backed lists (lists that sync with GitHub Issues), you must connect with the **Issues** scope. If you already linked GitHub for sign-in only, click **Reconnect for GitHub Issues** in the Connected Accounts section to grant the additional scope.
- **Mastodon** — Connect your Mastodon account. Once linked, you can select which Mastodon instances to cross-post to when composing a message.
- **Bluesky** — Connect your Bluesky account. Once linked, you can cross-post messages to Bluesky.
- **LinkedIn** — Connect your LinkedIn account for cross-posting (when LinkedIn is enabled on this instance).

Once a social account is linked, you can also use it to sign in. Cross-posting requires a **subscriber** account.

### Email Verification

You must verify your email before posting messages or attaching media. If you did not receive the verification email, click **Resend verification email** in the Security section. You can resend once every 10 minutes.

### Forgot Password

If you are locked out, use the **Forgot password** link on the login page. Enter your email address and a reset link will be sent.
